# Кнопка — лендінг голосування: перетворення прототипу на production-сайт

Дата: 2026-07-16

## Контекст і мета

Є готовий HTML-прототип (`knopka-vote-landing.html`) лендінгу голосування за продовження акції
«Рюкзак + в рюкзак» магазину «Кнопка». Лічильник голосів у прототипі працює через
`window.storage` — тимчасове сховище, доступне лише в Claude-чаті. Мета — перетворити прототип
на production-ready сайт на Next.js зі справжнім спільним лічильником голосів (Upstash Redis) і
підготувати його до деплою на Vercel.

Дизайн, копірайтинг і структура блоків лендінгу вже затверджені й **не змінюються** — переносяться
1:1 з прототипу.

## Архітектура

Next.js 14 (App Router, TypeScript), єдина сторінка:

- `app/layout.tsx` — базовий layout, підключення шрифтів, `<head>` метатеги.
- `app/page.tsx` — серверний компонент: статична розмітка (hero, умови акції, footer),
  `generateMetadata`/`metadata` для SEO.
- `app/VoteCard.tsx` — клієнтський компонент (`"use client"`): кільце-прогрес, кнопка голосування,
  конфеті, toast, live-оновлення лічильника (polling кожні 4с, як у прототипі).
- `app/globals.css` — стилі, перенесені 1:1 з `<style>` прототипу.
- `app/api/votes/route.ts` — API-маршрут:
  - `GET` → `{ count: number, voted: boolean }`
  - `POST` → інкрементує голос, повертає `{ count: number }` або помилку 429/403, якщо голос
    відхилено.

## Лічильник голосів (Upstash Redis)

- Пакет `@upstash/redis`, клієнт створюється з env-змінних `UPSTASH_REDIS_REST_URL` /
  `UPSTASH_REDIS_REST_TOKEN` (автоматично підтягуються Vercel Marketplace-інтеграцією).
- Ключ `votes:count`. Ініціалізація стартовим значенням **218** (як у прототипі) через
  `SETNX votes:count 218` при першому зверненні.
- Голосування — атомарний `INCR votes:count`, щоб уникнути race condition при одночасних голосах.

## Захист від накрутки голосів

Порядок перевірок у `POST /api/votes`:

1. **Cookie-перевірка** (найдешевша, без звернення до Redis): якщо в запиті є `httpOnly` cookie
   `knopka_voted=1` — одразу повертаємо 403, голос не рахується.
2. **Rate limit за IP**: беремо IP з `x-forwarded-for` (перший елемент), хешуємо (SHA-256, сирий IP
   не зберігається), перевіряємо існування ключа `ratelimit:<hash>` в Upstash Redis. Якщо є — 429.
   Якщо немає — ставимо ключ з TTL 24 години (`EX 86400`) і дозволяємо голос.
3. Якщо обидві перевірки пройдені — `INCR votes:count`, встановлюємо cookie `knopka_voted=1`
   (`httpOnly`, `maxAge` ~1 рік, `sameSite=lax`, `secure` в проді).

Фронтенд: при завантаженні сторінки робимо `GET /api/votes`, і якщо `voted: true` — кнопка одразу в
заблокованому стані (як у прототипі після голосування). Після успішного `POST` кнопка блокується без
перезапиту.

## Лого

Лишається CSS/SVG-імітація логотипу з прототипу без змін (оригінальний файл користувач надасть
пізніше окремим завданням — заміна на `<img src="/logo.png">` буде локальною зміною одного блоку в
`app/layout.tsx` або `page.tsx`, не архітектурною).

## SEO

Next.js Metadata API (`app/layout.tsx` / `app/page.tsx`): `title`, `description`, `og:title`,
`og:description` — перенесені з прототипу без змін. `og:image` не додається в цій ітерації — файл
зображення для соцмереж відсутній.

## Аналітика

- Google Analytics 4 та Meta Pixel підключаються через `next/script`, **умовно**: скрипт рендериться,
  лише якщо задана відповідна env-змінна (`NEXT_PUBLIC_GA_ID`, `NEXT_PUBLIC_META_PIXEL_ID`). Обидві
  змінні зараз порожні — користувач додасть значення в Vercel після деплою.
- Клік по кнопці голосування (тільки коли голос успішно зараховано) шле подію:
  `gtag('event', 'vote_click')` і/або `fbq('track', 'Lead')`, якщо відповідний скрипт підключено.

## Деплой

1. Локально: `git init`, перший коміт (вже виконано в цій сесії — репозиторій ініціалізовано в
   корені проєкту).
2. Користувач створює порожній репозиторій на github.com і надає Claude Code URL.
3. Claude Code додає `git remote`, пушить код.
4. Користувач самостійно: Vercel Import Project → підключення Upstash Redis через
   Storage → Marketplace Database Integrations → Deploy.
5. Піддомен `vote.knopka.shop` — **не налаштовується в цій ітерації**, користувачу поки достатньо
   стандартного `*.vercel.app` домену.

## Що НЕ входить у цю ітерацію

- `og:image` (немає файлу).
- Реальний файл лого (немає файлу, буде додано окремо).
- Значення GA4/Meta Pixel ID (користувач додасть після деплою через env-змінні у Vercel).
- Кастомний домен `vote.knopka.shop`.
