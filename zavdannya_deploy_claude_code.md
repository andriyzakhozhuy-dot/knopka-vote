# Завдання для Claude Code: перетворити прототип на робочий сайт і задеплоїти

## Контекст

Є готовий HTML-прототип лендінгу голосування за продовження акції "Рюкзак + в рюкзак" (файл `knopka-vote-landing.html`, додається). Дизайн, копірайтинг і структура вже затверджені — не змінюй їх без потреби. Задача: перетворити прототип на production-ready сайт із реальним спільним лічильником голосів і задеплоїти на Vercel.

**Важливо:** лічильник у прототипі працює через тимчасове артефакт-сховище (`window.storage`), яке існує лише всередині Claude-чату і не працюватиме на реальному сайті. Це треба замінити на справжній бекенд.

---

## Крок 1 — Ініціалізація проєкту

- Створити Next.js проєкт (App Router), оскільки він нативно деплоїться на Vercel і дозволяє додати серверні API-маршрути для лічильника (безпечніше, ніж звертатись до бази даних напряму з браузера).
- Перенести HTML/CSS/JS з прототипу в компоненти Next.js, зберігши весь дизайн, анімації, кольори, копірайтинг і структуру блоків без змін.

## Крок 2 — Реальний бекенд для лічильника голосів

**Важливо:** окремого продукту "Vercel KV" більше не існує — Vercel зняли його з лінійки. Тепер сховище типу Redis підключається через **Vercel Marketplace → Upstash Redis** (безкоштовний тариф є, підключення пряме з дашборду Vercel, credentials підтягуються автоматично як env-змінні).

Використати пакет `@upstash/redis` для роботи з базою.

Створити два API-маршрути:
- `GET /api/votes` — повертає поточну кількість голосів
- `POST /api/votes` — інкрементує лічильник на 1 (атомарна команда `INCR`, щоб уникнути race condition при одночасних голосах), з перевірками нижче

## Крок 3 — Захист від накрутки голосів

- При голосуванні виставляти `httpOnly` cookie `knopka_voted=1` з довгим терміном дії — при повторному запиті з тим самим cookie сервер відхиляє голос
- Додатково — rate limit за хешем IP у тій самій базі Upstash Redis: не більше 1 голосу з одного IP на 24 години
- На фронтенді кнопка одразу блокується після успішного голосування (як у прототипі)

## Крок 4 — Лого

У прототипі лого відтворено CSS/SVG-імітацією (наближено до оригіналу, не піксель-перфектно). Якщо є оригінальний файл лого (PNG/SVG з прозорим фоном) — покласти в `/public/logo.png` і замінити CSS-версію на `<img>`. Я додам файл лого окремо, якщо потрібно.

## Крок 5 — SEO і аналітика

- Мета-теги `title`, `description`, `og:title`, `og:description`, `og:image` (взяти з прототипу, доповнити og:image)
- Підключити Google Analytics 4 і/або Meta Pixel (event на клік по кнопці голосування — для трекінгу конверсій з реклами)

## Крок 6 — Деплой на Vercel

1. Запушити код у GitHub-репозиторій
2. Зайти на vercel.com → Import Project → вибрати цей репозиторій
3. У Vercel Dashboard → Storage → Marketplace Database Integrations → знайти **Upstash** → створити Redis-базу і підключити до проєкту (env-змінні `UPSTASH_REDIS_REST_URL` і `UPSTASH_REDIS_REST_TOKEN` додадуться автоматично)
4. Deploy
5. За бажання підключити піддомен, наприклад `vote.knopka.shop`: у Vercel → Settings → Domains додати домен, потім у панелі керування DNS основного домену `knopka.shop` додати CNAME-запис, який покаже Vercel

## Важливе застереження про безкоштовний план Vercel

Безкоштовний тариф Vercel (**Hobby**) за умовами використання призначений для **некомерційних** проєктів. Формально лендінг реального магазину — це вже комерційне використання, хоча на практиці для такого невеликого сайту з малим трафіком це ніхто не відстежує й не блокує. Якщо колись захочете убезпечитись повністю — це буде привід перейти на платний план Pro ($20/міс) або розглянути альтернативний хостинг, де комерційне використання дозволене на безкоштовному тарифі (наприклад Cloudflare Pages). Але для старту й тесту гіпотези з голосуванням — Hobby-плану Vercel цілком достатньо.
